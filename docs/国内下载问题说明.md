# 国内无法下载问题的解决过程说明

## 问题背景

原版方案依赖**在线下载**所有必要环境与依赖，包括：
- Ubuntu rootfs
- Node.js
- 系统基础包（apt）
- **OpenClaw 本体**（npm 安装）

在国内网络环境下，部分资源访问不稳定或无法访问，尤其是 **OpenClaw 本体** 从官方 npm 或 GitHub 下载经常失败，导致首次安装无法完成。

## 解决思路

1. **依赖镜像**  
   - 对 **npm**：在需要从网络安装 OpenClaw 时，将 registry 设置为国内镜像（如 `https://registry.npmmirror.com`），减少超时与连接失败。  
   - 对 **apt**：在实现上可配合重试与错误提示，必要时可替换为国内可用的软件源（具体以实际代码或文档为准）。

2. **将 OpenClaw 打包进 APK**  
   - 在构建 APK 前，在可访问外网的环境下**完整编译 OpenClaw**（含依赖）。  
   - 将编译结果打成 **openclaw.tgz**，放入 Flutter 工程的 `assets`（如 `assets/openclaw.tgz`）。  
   - 安装流程中**优先检测并解压该资源**到 rootfs 内对应目录，仅当资源不存在或无效时才回退到「npm 安装 + 国内镜像」。  
   - 这样在国内用户首次安装时，多数情况下**无需从外网下载 OpenClaw 本体**，仅需下载 rootfs、Node.js 等相对更容易获取的资源。

## 实现要点（与当前代码对应）

- **Bootstrap 流程**：先执行目录与 rootfs、Node 等步骤，再执行「安装 OpenClaw」。
- **安装 OpenClaw**：  
  - 若存在有效的 `assets/openclaw.tgz`，则解压到 rootfs 内全局 node 目录并创建 bin 包装器；  
  - 否则执行 `npm config set registry https://registry.npmmirror.com` 后执行 `npm install -g openclaw@latest ...`。
- 用户可见日志中会有「使用本地 openclaw.tgz 安装完成」或「从 npm 安装」等提示，便于排查问题。
