# DroidClaw 使用说明

## 首次使用

1. **安装 APK**  
   从 [Releases](https://github.com/iskandar510/DroidClaw/releases) 下载并安装。

2. **首次启动**  
   应用会引导完成环境安装（创建目录、下载/解压 Ubuntu rootfs、安装 Node.js、安装 OpenClaw 等）。部分版本会优先使用 APK 内自带的 OpenClaw，无需从外网下载。

3. **权限与优化**  
   - 建议在系统设置中为本应用关闭「电池优化」，避免后台被系统终止。  
   - 如需访问存储等，请按提示授予相应权限。

## 主要功能

### 仪表板
- 启动/停止/重启网关。
- 查看网关状态、健康检查。
- 快捷进入：终端、网关终端、Web 仪表板、日志、渠道、提供方、设置等。

### 终端
- 提供带 OpenClaw 的 Ubuntu 环境 shell。
- **常用命令列表**：内置一批 OpenClaw 相关命令（如设置本地模式、设置 Token、启动/停止/重启网关、查看配置、模型/渠道管理等），点击即可填入并执行。
- **自定义命令**：在设置或终端内可添加、编辑、删除自己的常用命令（显示名称 + 实际命令），方便一键执行。

### 网关令牌（Token）
- 路径：**设置 → 网关令牌**。
- 用于访问 Web 仪表板时的安全认证；可自定义或使用随机生成的令牌。

### OpenClaw 配置
- 路径：**设置 → 编辑 OpenClaw 配置**（或从设置进入「修改 openclaw 配置文件」）。
- 直接编辑 `root/.openclaw/openclaw.json`，支持 JSON 校验与保存。

### 其他设置
- 自动启动网关、电池优化、存储权限、可选包（Go、Homebrew、SSH）等，均在「设置」中管理。

## 使用流程简述

1. 完成首次环境安装。  
2. 在设置中（可选）设置「网关令牌」。  
3. 在仪表板启动网关。  
4. 通过「Web 仪表板」或终端中的 `openclaw` 命令进行模型、渠道等配置。  
5. 日常可在终端使用常用命令列表或自定义命令快速操作。
